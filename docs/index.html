Sentroy API

Complete reference for the Sentroy platform REST API.

Authentication

All API requests require a Bearer access token in the Authorization header. Create tokens from Admin → Access Tokens in your Sentroy dashboard.

Base URL (for cURL / raw HTTP)

https://sentroy.com/api/companies/{company-slug}

Replace {company-slug} with your company slug in all cURL endpoints below.

SDKs only need https://sentroy.com as the base URL — the /api/companies/{slug} path is built automatically from your companySlug config.

Quick Start

import { Sentroy } from "@sentroy-co/client-sdk"

const sentroy = new Sentroy({
  baseUrl: "https://sentroy.com",
  companySlug: "my-company",
  accessToken: "stk_...",
})
import sentroy "github.com/Sentroy-Co/client-sdk/go"

client := sentroy.New(sentroy.Config{
    BaseURL:     "https://sentroy.com",
    CompanySlug: "my-company",
    AccessToken: "stk_...",
})
from sentroy import Sentroy

sentroy = Sentroy(
    base_url="https://sentroy.com",
    company_slug="my-company",
    access_token="stk_...",
)
use Sentroy\ClientSdk\Sentroy;

$sentroy = new Sentroy([
    'base_url'      => 'https://sentroy.com',
    'company_slug' => 'my-company',
    'access_token' => 'stk_...',
]);

Error Handling

Error responses use a consistent envelope:

{
  "data": null,
  "error": "Human-readable error message"
}
StatusMeaning
400Bad request — missing or invalid parameters
401Invalid or expired access token
403Insufficient permissions or token/company mismatch
404Resource not found
500Internal server error

POST /send

  -H "Authorization: Bearer stk_..." \
  -H "Content-Type: application/json" \
  -d '{
    "to": "user@example.com",
    "from": "info@example.com",
    "subject": "Welcome!",
    "domainId": "a1b2c3d4-...",
    "templateId": "b3f1a2c4-...",

const result = await sentroy.send.email({
  to: "user@example.com",
  from: "info@example.com",
  subject: "Welcome!",
  domainId: "a1b2c3d4-...",
  templateId: "b3f1a2c4-...",

})

Buckets

Isolated storage containers with their own visibility (public vs private) and usage counters. Storage requests use the /api/storage/companies/{company-slug} base URL — the same access token that signs mail calls.

List buckets

GET /buckets
curl -s https://sentroy.com/api/storage/companies/{company-slug}/buckets \
  -H "Authorization: Bearer stk_..."
const buckets = await sentroy.buckets.list()

Get bucket

GET /buckets/{bucket-slug}
curl -s https://sentroy.com/api/storage/companies/{company-slug}/buckets/public \
  -H "Authorization: Bearer stk_..."
const bucket = await sentroy.buckets.get("public")

Create bucket

POST /buckets
curl -s -X POST https://sentroy.com/api/storage/companies/{company-slug}/buckets \
  -H "Authorization: Bearer stk_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Public assets",
    "slug": "public",
    "isPublic": true
  }'
await sentroy.buckets.create({
  name: "Public assets",
  slug: "public",
  isPublic: true,
})

If slug is omitted the server derives one from name.

Update bucket

PATCH /buckets/{bucket-slug}
curl -s -X PATCH https://sentroy.com/api/storage/companies/{company-slug}/buckets/public \
  -H "Authorization: Bearer stk_..." \
  -H "Content-Type: application/json" \
  -d '{ "isPublic": false }'
await sentroy.buckets.update("public", { isPublic: false })

Toggling isPublic cascades to every existing file's S3 ACL and Media record — large buckets can take a while.

Delete bucket

DELETE /buckets/{bucket-slug}
curl -s -X DELETE "https://sentroy.com/api/storage/companies/{company-slug}/buckets/public?force=true" \
  -H "Authorization: Bearer stk_..."
await sentroy.buckets.delete("public", { force: true })

Without force=true, a non-empty bucket returns 409. With the flag set, every file is purged from S3 and the Media records are removed before the bucket itself is dropped.

Media

Files inside a bucket. Uploads are multipart; images receive a thumbnail ladder automatically.

List files

GET /buckets/{bucket-slug}/media
curl -s "https://sentroy.com/api/storage/companies/{company-slug}/buckets/public/media?type=image&limit=50" \
  -H "Authorization: Bearer stk_..."
const page = await sentroy.media.list("public", {
  type: "image",
  limit: 50,
})

Query params: type (image | video | audio | document | other), folder, limit (default 50, max 200), skip.

Get file

GET /buckets/{bucket-slug}/media/{media-id}
curl -s https://sentroy.com/api/storage/companies/{company-slug}/buckets/public/media/65f8a1... \
  -H "Authorization: Bearer stk_..."
const file = await sentroy.media.get("public", "65f8a1...")

Upload file

POST /buckets/{bucket-slug}/media
curl -s -X POST https://sentroy.com/api/storage/companies/{company-slug}/buckets/public/media \
  -H "Authorization: Bearer stk_..." \
  -F "file=@./photo.jpg" \
  -F "folder=marketing" \
  -F "public=true" \
  -F "alt=Hero shot" \
  -F "tags=hero,landing"
const media = await sentroy.media.upload("public", {
  body: file,
  folder: "marketing",
  isPublic: true,
  alt: "Hero shot",
  tags: ["hero", "landing"],
})
console.log(media.url)

Multipart fields: file (required), folder, public, alt, caption, tags (comma-separated). Public files are accepted only when the bucket itself is public. Image uploads return thumbnail variants under imageMeta.thumbnails.

Download file

GET /buckets/{bucket-slug}/media/{media-id}/download
curl -s -L "https://sentroy.com/api/storage/companies/{company-slug}/buckets/public/media/65f8a1.../download?quality=500" \
  -H "Authorization: Bearer stk_..." \
  -o photo-500.jpg
const blob = await sentroy.media.download(
  "public",
  "65f8a1...",
  { quality: 500 },
)

For private buckets this is the only way to fetch the bytes — the public CDN URL refuses them. Pass quality to request a pre-generated thumbnail variant; falls back to the original if that size doesn't exist.

Delete file

DELETE /buckets/{bucket-slug}/media/{media-id}
curl -s -X DELETE https://sentroy.com/api/storage/companies/{company-slug}/buckets/public/media/65f8a1... \
  -H "Authorization: Bearer stk_..."
await sentroy.media.delete("public", "65f8a1...")

Cascades through the CDN: S3 objects (original + every thumbnail) are removed before the Media record. If any S3 delete fails the record is kept so you can retry without losing the reference.

Storage usage

Read-only quota and breakdown endpoints — no auth scope beyond a valid access token. Useful for usage dashboards.

Storage quota

GET /storage-quota
curl -s https://sentroy.com/api/storage/companies/{company-slug}/storage-quota \
  -H "Authorization: Bearer stk_..."
const quota = await sentroy.storage.quota()

limit: 0 means the plan is unlimited. used reflects every bucket's total; mailUsed is the mail product's share of the same plan pool.

Usage breakdown

GET /usage
curl -s https://sentroy.com/api/storage/companies/{company-slug}/usage \
  -H "Authorization: Bearer stk_..."
const usage = await sentroy.storage.usage()
// usage.quota, usage.buckets, usage.byType

Single round-trip — quota + per-bucket counters + media-type aggregation in one payload. Intended for dashboards.
